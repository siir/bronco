Summary
Recurring deadlock detected on evo_EDI_Document table during nightly EDI batch processing. The deadlock involves two concurrent update operations on the same clustered index. Average frequency: 3-4 times per batch run. Impact: batch processing delays of 15-20 minutes.
AI Analysis
DEEP_ANALYSIS claude-sonnet-4-5 · 4,218 tokens · 3.2s
Root cause is a missing covering index on the EDI document staging table. The batch processor performs UPDATE...FROM with a self-join that escalates to table-level locks. Recommended: add a non-clustered index on (DocumentId, ProcessingStatus) INCLUDE (BatchId, UpdatedAt) and refactor the UPDATE to use row-level locking hints.
